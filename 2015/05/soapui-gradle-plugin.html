SoapUI Gradle Plugin

14 May 2015

Tags: gradle plugin soapui

This one nearly ended me!

For quite some time I have been trying to write a Gradle plugin for the popular SoapUI tool. On first impressions I thought this was a simple port from the Maven version. Oh how I was wrong!!

First there was the wonderful classpath hell scenario that Renato Athaydes described in his [blog](https://sites.google.com/a/athaydes.com/renato-athaydes/posts/writingagradlepluginforsoapui). This was a really painful exercise and I’m surprised given the number of issues that this wasn’t an urgent issue for SmartBear. However, with some perseverence and a big thanks to the information Renato captured I was able to close this one out.

Then came the [logging issues..](https://discuss.gradle.org/t/classpath-hell-soapui-and-gradle-api-logging-conflicts/8830). turns out there was a conflict between Gradle and SoapUI.
